# UI 大幅改修 デザインレビュー記録

Issue #19 / branch `claude/hopeful-cray-mabkoj`。

目的: 「AI が作った UI 感」の払拭。
1. lucide-react のアイコン採用（全絵文字を撤廃）
2. 付箋（EventStorming 7色）以外の配色を、楽しく一貫したブランドパレットに統一
3. UI/UX デザイナー視点のレビューを 5 回以上回して反映

## 不変条件（レビューでも侵さない）
- EventStorming カード7色は厳密一致（変更禁止）。装飾に流用しない。
- 三重冗長（色＋アイコン＋ラベル）を維持。
- `src/game` / `src/data` は UI 非依存（アイコンは意味トークン、解決は UI 層）。
- WCAG AA・色覚多様性・reduced-motion を維持。

各ラウンドは実画面のスクリーンショットを観察 → 指摘 → 反映、の順で進めた。

---

## ラウンド1 — 全体クラフト（視覚階層・配色ロジック・アイコン枠）

観察: 全7画面のスクリーンショット。

主な指摘と対応:
- **P1 オンボーディングが素の暗い背景でタイトルが浮いていた**（最初の画面が一番作り込まれていない=AI感）→ `.onboard-hero` を `--grad-joy` のブランドパネル化（やわらかい光・影付き）。
- **P1 アイコンの「枠あり/なし」が混在**（node-icon は枠付き、chapter/play は素）→ 章タイトル・プレイ見出しのアイコンを node-icon と同じ枠付きタイルに統一。
- **P1 報酬の金（--reward）が報酬以外にも使われ配色ロジックが濁る**→ 金は「獲得・達成」専用（星・達成数値・章クリア・NEW語彙）。情報アイコン（実務解説・成功メッセージ・NEXT CHAPTER）はアクセント側へ。
- **P1 完走トロフィーが「ただの円」**→ 金のハロー＋淡いインナーリングで山場感を付与。
- **P1 ヒーローとappbarで StormQuest が二重**→ ヒーローに固有のタグライン（価値提案）を追加して内容差を作る。
- **P2 図鑑見出しが章タイトルと同階層**→ 図鑑をオーバーライン（小・字間広め・uppercase）にして階層を分離。

不変条件は維持（カード7色・三重冗長・AA・reduced-motion）。

## ラウンド2 — ゲームプレイ画面のクラフト

観察: 03 タイムライン / 04 凡例 / 06 定義シート。

主な指摘と対応:
- **P1 空スロットが「貼れる場所」に見えない**→ 空のスロット行を凹ませ（点線＋淡いアクセント地＋内側影）、Target アイコン＋文言で誘導。配置後は実線＋白板に切替（点線=未確定 / 実線=確定 を MODE2 と統一）。
- **P1 時間軸（過去/未来）が float で素**→ 過去↔未来を1本のグラデ・レールで結ぶ time-axis に再構築。
- **P1 ヒントと主CTAが同じ全幅で階層が曖昧**→ ヒントを幅詰め・中央寄せ・caption サイズの副次アクションに降格。主CTAを明確化。
- **P2 「説明と記法」の開閉が捨て見出し**→ 面（surface-sunken）＋先頭/シェブロンのアイコン付きタップ行に。開時にシェブロン回転、タップ標的 44px。
- **P2 定義シートの×が極小**→ 40px の丸ヒットエリアに。シートの max-height を 40vh→min(72vh,560px) に上げ本文が隠れないように。
- **P2 手札ラベルが弱い**→ サイズ・色を上げ「ゾーン見出し」として認識できるよう調整。

不変条件は維持。

## ラウンド3 — トリガー/不変条件モード ＋ アクセシビリティ

観察: 08 トリガー / 09 不変条件ゲート / 03 比較 / コントラスト実測。

主な指摘と対応:
- **P1 MODE2 の空スロットがタイムラインの凹みドロップゾーンと別言語**→ `.trigger-slot` 空状態を accent-tint＋内側影に統一、接続済みは実線＋白板へ（点線=未確定 / 実線=確定 を全モード統一）。
- **P1 `--ink-faint`(#9a93bd ≈3.0:1) が本文・ラベルに使われ AA 不達**→ 定義シートの英名・スロット番号・ロック表示・図鑑空きを `--ink-muted`(≈4.9:1) に。`--ink-faint` は非テキスト装飾専用へ。
- **P1 金(#f5a623)は白地で ≈1.9:1**→ 星は塗り/抜きで意味を担保した上で、白地のエッジ用に濃い輪郭 `--reward-ink` を追加。
- **P2 不変条件チェックリストの緑✓が「充足済み」に誤読**→ 中立マーカー `›` に変更。緑✓は判断後の gate-feedback.ok 専用に（学習の誠実さ・②）。
- **P2 不変ゲートの階層**→ 「判定対象のコマンド」を影＋余白で前面化し、参照の集約パネルは静かな文脈に。
- **P2 ヒントボタンの枠が薄く浮く**→ 面（surface-sunken）で塗り、副次のピルとして明確化。

不変条件は維持。

## ラウンド4 — タイポ階層・余白リズム・ダーク画面・デスクトップ

観察: 10 リザルト / 07 完走 / 11 デスクトップ。

主な指摘と対応:
- **P1 `--fs-display` が見出しと統計数値で兼用＝主役不在**→ `--fs-display-lg`(2.125rem) を新設しリザルト/完走タイトルへ。stat より明確に大きく。
- **P1 ダーク画面の半透明パネルが全て同一の角丸矩形＝量産感**→ reality は上ハイライト＋アクセント左罫の「余談」、next-chapter は点線の「予告」に役割分化。
- **P1 学習の主役テキスト（語彙定義・実務解説）が caption 級で小さい**→ `--fs-body` に拡大（DESIGN.md ①②の優先順位に整合）。
- **P1 リズムが全部 space-lg で均一**→ 報酬クラスタ（語彙＋解説）を近接させ CTA と大きく離す。
- **P2 grad-joy の使い回し過多**→ 完走トロフィーを放射状グラデ＋金リングのトロフィー固有表現に。
- **P2 デスクトップが「ただの細い列」**→ 外周シェルにやわらかな放射光を敷きフレームを“置かれたアプリカード”に。
- **P2 章クリア帯が主役の星バーストと競合**→ ピル・小サイズに降格。
- **P2 on-dark ゴーストボタンの枠が薄い**→ 透明度 .4→.55 に。

不変条件は維持。

## ラウンド5 — 最終ホリスティック（サインオフ）

観察: 全10画面（モバイル＋デスクトップ）。

総評（デザイナー）: 「**AI生成の雰囲気は払拭された。サインオフ品質**。紫→ピンクのヒーロー、金の報酬アクセント、枠付き章アイコン、凹み/確定のドロップゾーン、一貫した日本語タイポが1つの意図あるブランド系として噛み合っている」。

ゴール達成チェック（デザイナー判定）:
- lucide-react アイコン採用 … MET（全アイコンが `Icon.tsx` 経由で解決、描画に生グリフなし）
- 絵文字撤廃 … MET（描画 UI に絵文字ゼロ）
- 付箋以外の配色の統一 … MET（紫/ピンク＋金をヒーロー・進捗・星・統計・章バッジで一貫適用。カード7色は不変）

残課題と対応:
- **P2 ヒーローの進捗テキストがピンク地で白小文字＝AA 際どい**→ ヒーロー文字に薄い影＋pct を白・太字に。
- 現在ノードのシェブロン／トリガー空スロットの凹み表現は既に適用済み（指摘は旧版由来）と確認。
