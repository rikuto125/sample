# CONTEXT — ユビキタス言語（JA↔EN）

StormQuest が教える EventStorming の用語集。**これが正**。新しい用語は先にここへ追記してから実装する。

## EventStorming 記法カード

| JA | EN | 色（標準） | アイコン | 定義 |
|---|---|---|---|---|
| ドメインイベント | Domain Event | オレンジ | ⚡ | ビジネス上で「過去に起きた事実」。必ず過去形（〜した／〜された）。 |
| コマンド | Command | 水色 | ▶ | システムへの指示・意図。「〜する」と現在形。イベント（事実）とは時制で見分ける。 |
| アクター | Actor | 黄(小) | 👤 | コマンドを起こす人・役割（顧客、店長…）。 |
| ポリシー | Policy | 紫 | ⚙ | 「◯◯したら、△△する」— イベントに反応して次のコマンドを自動で起こすルール。 |
| 外部システム | External System | ピンク | ☁ | 自分たちの管理外のシステム（決済ゲートウェイ、地図API…）。 |
| リードモデル | Read Model | 緑 | 📋 | 人が意思決定するために参照する情報（画面・一覧）。イベントから導かれる。 |
| 集約 | Aggregate | 淡黄(大) | 📦 | 不変条件を守る一貫性の境界（第2章以降）。 |

## ゲーム内概念

| JA | EN | 定義 |
|---|---|---|
| ステージ | Stage | 1つの学習単位。MODE1（タイムライン）か MODE2（トリガー接続）のいずれか。 |
| タイムラインモード | Timeline Mode | ドメインイベントを時系列に並べる。半順序制約で採点（唯一解を強制しない）。 |
| トリガー接続モード | Trigger Mode | 各コマンドに正しいトリガー（Actor/Policy/External）をつなぐ。複数正解を許容。 |
| ダミーカード | Distractor | MODE1 で混ざる「イベントでないカード」。弾けると理解の証。 |
| 因果連鎖 | Causal Chain | Event → Policy → Command → Event。EventStorming の本丸。最終ステージで体験させる。 |
| 半順序制約 | Partial-Order Constraint | 「A は B より前」という前後関係の集合。全順序を強制しない採点の核。 |
| 用語図鑑 | Vocabulary Codex | 獲得した語彙のコレクション。空きスロットが学習動機になる。 |
| 実務リアリティ解説 | Reality Note | クリア後に出る「現場ではこうなっている」という解説。 |
| 孤立コマンド | Orphan Command | 起動経路（トリガー）が宙吊りのコマンド。これをゼロにするのが Phase 8 の正しい不変条件。 |
